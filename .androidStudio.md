# Configuración de Android Studio en WSL (Windows Subsystem for Linux)

Para compilar la APK localmente usando `eas build --local` desde WSL apuntando al SDK de Windows, se deben seguir estos pasos:

### 1. Variables de Entorno
Añadir estas líneas al archivo de configuración de tu shell (`~/.zshrc` o `~/.bashrc`):

```zsh
export ANDROID_HOME=/mnt/c/Users/Hugop/AppData/Local/Android/Sdk
export PATH=$PATH:$ANDROID_HOME/platform-tools
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/cmdline-tools/latest/bin
```

### 2. Configuración del Proyecto
Dentro de la carpeta `tiempo-app`, crear el archivo `local.properties` para Gradle:

```bash
mkdir -p android && echo "sdk.dir=/mnt/c/Users/Hugop/AppData/Local/Android/Sdk" > android/local.properties
```

### 3. Comando de Compilación
Para compilar sin errores de repositorio Git (si estás en una subcarpeta) y forzando la ruta del SDK:

```bash
ANDROID_HOME=/mnt/c/Users/Hugop/AppData/Local/Android/Sdk EAS_NO_VCS=1 eas build --platform android --local --profile production
```

---
*Nota: Asegúrate de que la ruta `/mnt/c/Users/Hugop/AppData/Local/Android/Sdk` es accesible desde WSL con `ls`.*
